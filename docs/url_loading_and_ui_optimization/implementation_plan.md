# 実装計画: URL直接読み込み機能およびUI・名称の包括的最適化

本計画は、ユーザーからのURL指定によるオンラインPDF読み込み機能の実装、CORS制限を突破するプロキシフォールバック通信の確立、およびUI/UXの抜本的改善（フォント化け撲滅・60FPSサムネイル拡縮）とアプリ名統一の全工程をまとめたものです。

## 課題と目標
1. **URLダウンロードの実現**: `ehttp` と `urlencoding` を導入し、WASM / Native の両環境でURLから直接PDFを非同期ダウンロード可能にする。
2. **CORSおよびWAFの突破**: WASM環境からの外部リクエストに対し、複数プロキシを自動巡回・再試行するフォールバックチェーンを実装する。特に官公庁サイト等で返されるWAFエラー画面（HTML）を `%PDF` マジックナンバーで検知し、フリーズを防ぐ。
3. **日本語UIの完全美化**: Web版でフォントが同梱されていないことによる「豆腐」を防ぐため、アプリ起動時に `jsdelivr` CDNから軽量なM+ PLUS 1pフォントを非同期取得し、確実な同期タイミングで適用する。
4. **極上のサムネイル拡縮UX**: サムネイルのサイズスライダー操作時にキャッシュを破棄せず、GPUのリアルタイム拡縮を活用することで、60FPSで滑らかにサイズが変化するUXを実現する。
5. **名称統一**: すべての環境でアプリ名を「PDF Page to Clipboard」に統一する。

## 実装内容
* `Cargo.toml`: ネットワーク通信用クレートの追加
* `src/main.rs`: 通信ロジック、マジックナンバー検証、フォント同期ループ、GPU拡縮ロジックの統合
* `index.html`: Web版のタイトル統一
