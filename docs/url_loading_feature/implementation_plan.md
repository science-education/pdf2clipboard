# URLからのPDF読み込み機能 実装計画

## 概要
ユーザーがWeb上のPDFのURLを入力し、直接ダウンロードして表示・コピーできる機能を追加します。
Windows、Mac、およびWASM（Webブラウザ）のすべてのターゲットでシームレスに動作するようにします。

## ユーザーレビューが必要な事項 / 提案
* **CORS プロキシについて (WASM版)**:
  Webブラウザの仕様上、外部のサーバー（官公庁や大学のサイトなど）からPDFをJavaScript/WASM経由で取得する際、サーバー側がCORS（異なるドメインからのアクセス許可）を設定していないとエラーになります。
  そのため、ダウンロードに失敗した場合は自動的にパブリックなCORSプロキシ（例: `https://corsproxy.io/?`）を経由して再試行する堅牢な仕組みを提案します。

## 提案する変更

### 1. 依存関係の追加 (`Cargo.toml`)
* `ehttp` クレート（v0.5）を追加。egui と親和性が高く、Native と WASM の両方で同一の非同期HTTPリクエストAPIを提供します。

### 2. UI と状態の拡張 (`src/main.rs`)
* **構造体 (`App`)**:
  ```rust
  url_input: String,
  is_downloading: bool,
  ```
* **トップバーUI**:
  ステータス表示の横または上部に、URL入力用のテキストボックス (`TextEdit`) とダウンロード開始ボタン（「開く / Load」）を配置。

### 3. 非同期ダウンロード処理の実装
* ボタン押下時またはEnterキー押下時に `ehttp::fetch` を実行。
* ダウンロード中は UI を「ダウンロード中...」のステータス表示（またはスピナー）に切り替え。
* 取得したレスポンス（バイト列）を既存の `load_pdf_bytes` へ非同期に渡して画面を更新。

## 検証計画
* **Native (Windows)**: `cargo run` で起動し、オンラインの PDF URL (例: arXiv やサンプルのPDF URL) を入力して正常にサムネイルが表示されるか検証。
* **WASM (Web)**: `trunk serve` で起動し、同様に URL から読み込めるか検証（CORS プロキシのフォールバック動作も確認）。
