# サムネイル操作の視覚効果およびステータス表示の改善計画

サムネイルのホバー時およびクリック（コピー）時のユーザーエクスペリエンスを向上させるため、視覚的なフィードバックを強化します。

## 課題の現状
- ホバー時はテキストツールチップのみが表示される。
- クリック時は即座に赤枠が表示されるが、実際のコピー処理（特に高解像度レンダリング時）の進捗が分かりにくい。

## 提案する変更内容

### 1. `PageSlot` への状態管理フィールドの追加
- 各ページのコピー状態を管理するため、以下の列挙型を導入し `PageSlot` に追加します。
  ```rust
  #[derive(Clone, Copy, PartialEq)]
  enum CopyState {
      Idle,
      Copying,
      Done,
  }
  ```

### 2. コピー完了通知メカニズムの導入
- バックグラウンドスレッドでのコピー完了をメインスレッド（UIスレッド）に伝えるため、`mpsc` チャネルまたは共有状態を使用します。
- メインスレッドの `update` ループ内でこの通知を受け取り、該当ページの `CopyState` を `Done` に更新します。

### 3. UI 描画ロジックの刷新 (`src/main.rs`)
- **ホバー時**: `resp.hovered()` を検知し、サムネイルの矩形全体に半透明の赤 (`Color32::from_rgba_unmultiplied(255, 0, 0, 100)`) を `painter().rect_filled()` で描画します。
- **コピー開始時**: クリックされたページの `CopyState` を `Copying` に設定し、既存の赤枠を非表示（またはクリア）にします。
- **オーバーレイ表示**:
  - `Copying` 状態: サムネイル中央に「COPY中」というテキストを白文字で描画。
  - `Done` 状態: サムネイル中央に「完了」というテキストを描画し、さらに外側に赤枠を描画。
- **既存のツールチップ削除**: `on_hover_text` を削除します。

### 4. `copy_page` 関数の修正
- コピー処理の開始直前に状態を `Copying` に変更。
- スレッド内での `clipboard_set_windows` 完了後、メインスレッドに完了通知を送り、`ctx.request_repaint()` を呼び出します。

## ユーザーレビューが必要な事項

> [!NOTE]
> 「完了」表示と赤枠は、別のページがコピーされるまで維持する方針でよろしいでしょうか。それとも一定時間（例: 2秒）で消えるようにしますか？今回は一旦、別のページが選択されるまで維持する（現在の赤枠の挙動に近い）形で実装します。

## 検証計画

### 手動検証
1. サムネイルにマウスを重ね、半透明の赤に変わることを確認。
2. クリックし、即座に「COPY中」と表示されることを確認。
3. コピー完了後（高解像度の場合は数秒後）に「完了」と表示され、赤枠がつくことを確認。
4. クリップボードに正しく画像がコピーされていることを確認。
