# 多言語化（日本語・英語）対応の実施計画

OSの言語設定を自動的に判別し、UIのテキスト（メッセージ、ラベル、ステータス表示など）を日本語または英語に切り替える機能を実装します。

## 課題
- 現在、UIのテキストに日本語と英語が混在しており、多言語対応がなされていない。
- ユーザーの環境（OS言語）に合わせて適切な言語を表示する必要がある。

## 変更内容

### 1. 依存関係の追加 (`Cargo.toml`)
- 言語設定を取得するため、軽量な `sys-locale` クレートを追加します。
  ```toml
  sys-locale = "0.3"
  ```

### 2. 翻訳定義の導入 (`src/main.rs`)
- 翻訳情報を保持する構造体 `Tr` を定義し、日本語用 (`TR_JP`) と英語用 (`TR_EN`) の定数を作成します。
- 対象テキスト：
  - 「PDFファイルをここにドロップしてください」 / "Drop a PDF file onto this window"
  - 「DPI:」 / "DPI:"
  - 「サムネイル:」 / "Thumbnail:"
  - 「COPY中」 / "Copying..."
  - 「完了」 / "Done"
  - ステータスバーの各種メッセージ（コピー完了、レンダリング中など）

### 3. アプリ起動時の言語検知
- `App::new` 内で `sys_locale::get_locale()` を実行。
- 取得したロケール文字列が `"ja"` で始まる場合は日本語モード、それ以外は英語モードとして初期化します。

### 4. UI 描画およびステータス更新の修正
- ハードコードされている文字列を `self.tr.xxx` の形式に置き換えます。
- 動的なメッセージ（ページ番号を含むものなど）については、翻訳定義側でフォーマットを工夫するか、その場で言語判定を行って構築します。

## ユーザーレビューが必要な事項

> [!NOTE]
> ロケールの取得に失敗した場合は、デフォルトで英語を表示する仕様とします。

## 検証計画

### 手動検証
1. 日本語環境の Windows で起動し、UIが日本語（「PDFファイルを〜」等）になっていることを確認。
2. （可能であれば）言語設定を英語に変更して起動し、UIが英語に切り替わっていることを確認。
3. コピー操作を行い、「COPY中」「完了」のテキストが設定言語に合っていることを確認。
